## CDK Infrastructure Rules (TypeScript)

AWS CDK infrastructure-as-code for the Envilder project. Deploys the static
website (Astro) and CI/CD pipeline.

### Structure

- `app/main.ts` — Entry point. Creates adapters and injects into handler
- `app/deployment.config.ts` — Top-level deployment config (repo, domain, stacks)
- `domain/` — Pure models, ports, errors, and validation
- `application/` — Use case handler (DeployInfrastructureHandler)
- `infrastructure/` — Adapters: logging, path resolution, CloudFormation utils, CDK stacks

### Conventions

- **Declarative config** — Define resources in config objects, not imperative CDK code
- **Environment-aware**: Use `AppEnvironment` enum (Development, LocalDevelopment, Test, Production)
- **Dependency Injection** — Inject ports (ILogger, IProjectPath) into handler; never instantiate adapters in application layer
- Import from local `config/` directory — no external shared-iac dependency

### Config Rules

- One config file per layer (`frontend`, `shared`)
- All timeouts and limits must have explicit values (no CDK defaults)

### Testing

- Snapshot tests in `tests/iac/` — update snapshots when stack changes are intentional
- Run `pnpm test` from tests/iac directory after any config or construct change

### Formatting

- Biome: 4 spaces, double quotes, semicolons, trailing commas, LF
- `import type` for type-only imports
